# /pick-persona — 페르소나 후보 좁히기

> 사용 시점: 단계 3 (페르소나 선택). `/welcome` 끝나고 호출.

## 갱이가 할 일

### 1. 5문 취향 인터뷰

한 번에 하나씩 묻고, 사용자 답을 기다린다. *대신 골라 주지 않는다.*

1. "최근 본 영화·드라마·만화 중 인상 깊었던 캐릭터 한 명만 알려 주실래요?"
2. "그 캐릭터의 *말투*가 좋았어요, 아니면 *생각하는 방식*이 좋았어요?"
3. "갱이처럼 매번 같은 톤이 좋아요, 아니면 상황마다 톤이 변하는 게 좋아요?"
4. "응답이 짧고 명료한 게 좋아요, 아니면 비유·인용 많은 게 좋아요?"
5. "절대 안 했으면 하는 게 있나요? (예: 욕설·정치·존댓말 깨지는 것)"

### 2. 후보 3개 추천

5문 답을 종합해 *서로 다른 갈래*로 3개 추천. 예시 카드:

```
후보 A — 영화 캐릭터:  [이름]
  핵심: [한 줄 톤]
  추천 이유: [사용자 답변과 연결]

후보 B — 콘셉트 페르소나:  [예: 새벽 라디오 DJ]
  핵심: [한 줄 톤]
  추천 이유: [사용자 답변과 연결]

후보 C — 본인 톤 (W7 미리 맛보기):
  핵심: 본인 평소 말투
  추천 이유: [사용자 답변과 연결]
```

### 3. 결정 위임

"이 중 하나 골라 주시면 갱이가 `/research-persona` 로 시스템 프롬프트 초안 만들어 드릴게요 🐾"

## 금기

- 후보 4개 이상 늘리지 않기 (선택 피로)
- 사용자 대신 결정하지 않기
- 회사 코드명·제3자 실명은 후보에서 제외 (CLAUDE.md §5)
