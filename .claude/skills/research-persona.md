# /research-persona — 시스템 프롬프트 초안 작성

> 사용 시점: 단계 4 (시스템 프롬프트 작성). `/pick-persona` 끝나고 페르소나가 정해진 후.

## 갱이가 할 일

### 1. 페르소나 정보 모으기 (가벼운 리서치)

- WebSearch 또는 사용자가 추가로 알려준 정보 활용
- 캐릭터라면: 출처 작품·핵심 대사·말투 특징
- 콘셉트라면: 사용자가 정의한 룰
- 본인 톤이라면: 사용자에게 *최근에 보낸 메시지 3~5개*를 요청해서 분석

### 2. `persona/system_prompt.md` 작성

`persona/system_prompt.md.example` 구조를 그대로 따른다:

```markdown
# [페르소나 이름] 시스템 프롬프트

당신은 **[페르소나 이름]** 입니다.

## 정체성
- [한 줄]
- [출처 1줄]

## 톤 키워드 3개
- [키워드 1]
- [키워드 2]
- [키워드 3]

## 자기 호칭
- [한 단어]

## 첫 인사
- [1~2줄]

## 금기 (1줄)
- [한 줄]

## 응답 스타일
- 길이: [짧고 명료 / 중간]
- 이모지: [쓸지 안 쓸지]
- 비유·인용: [좋아하는지]
```

### 3. 길이 체크

전체 분량이 **200~400자** 사이인지 갱이가 확인. 너무 짧으면 톤이 안 살고, 너무 길면 모델이 무거워져요.

```bash
wc -m persona/system_prompt.md
```

### 4. 다음 단계 안내

"초안 박았어요. 텔레그램 띄우기 전에 `/simulate` 로 5문답 미리 굴려 봐요 🐾"

## 금기

- 사용자 동의 없이 페르소나 정체성을 임의로 바꾸지 않기
- 회사 코드명·제3자 실명을 페르소나 출처로 쓰지 않기 (CLAUDE.md §5)
- 민감 정치·종교 발언 권장 페르소나 작성 금지
