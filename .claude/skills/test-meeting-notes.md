# /test-meeting-notes — 샘플 메모로 첫 회의록 시연

> 사용 시점: W5 step 3 (첫 시연) + step 6 (페어 시연).
> 봇이 살아 있고 `config/meeting_notes.yml` 이 있으면 바로 가능해요.

## 갱이가 할 일

### 1. config 준비 확인

```bash
test -f config/meeting_notes.yml || cp config/meeting_notes.yml.example config/meeting_notes.yml
grep -E 'enabled|length|action_item_format' config/meeting_notes.yml
```

`.gitignore` 에 박혀 있어서 안심하고 채워도 OK.

### 2. 샘플 회의 메모 띄우기

repo 안에 시연용 가상 회의 메모가 있어요 (실제 정보 아님):

```bash
cat docs/sample-meeting-notes-ko.txt
```

사용자에게 안내:

> "이 메모를 통째로 복사해서, 텔레그램 본인 봇에 *한 메시지로* 붙여 넣어 보세요.
>  앞에 `/notes` 를 붙여도 되고, 그냥 메모만 붙여도 자동으로 회의록 모드로 인식해요 🐾"

### 3. 두 가지 시연 방법 (아무거나)

```
방법 A — 명령으로:
   /notes
   (줄바꿈 후 메모 통째로 붙여넣기, 또는 /notes 뒤에 바로 붙이기)

방법 B — 자연어로:
   메모를 붙여 넣은 뒤 끝에 "회의록으로 정리해줘" 추가
   또는 긴 메모(600자 이상·여러 줄)는 그냥 보내도 자동 인식
```

### 4. 도착 확인 — 무엇이 와야 하나

5~30 초 안에 본인 텔레그램에 회의록이 도착해야 해요:

```
(yml 에 ack_message 박았다면) 즉시 ack 한 줄
   ↓
## 한 줄 요약
## 핵심 논의
## 결정사항
## 액션아이템   (- [ ] 할 일 — 담당자 · 기한)
## 미해결 질문
```

> 회의록이 길면 *여러 메시지로 나뉘어* 와요 — (1/2), (2/2) 처럼 표시돼요. 정상이에요 🐾

### 5. 사실 보존 확인 — 가장 중요한 점검

사용자에게 묻기:

> "회의록을 원본 메모와 *한 번 대조*해 보세요:
>  - 날짜·기한(6/24, 6/27, 6/20)이 그대로인가요?
>  - 참석자 이름(지수·민호·서연·태경)이 안 바뀌었나요?
>  - 결정사항이 *실제로 결정된 것만* 들어갔나요? (A/B 테스트는 '안 하기로' 결정 → 결정사항/미해결 어디에 들어갔는지 확인)
>
>  페르소나는 *톤만* 입히고 사실은 안 바꿔요. 만약 사실이 틀어졌으면 갱이한테 알려 주세요 — 시스템 프롬프트 점검할게요."

### 6. **여기까지 됐나요?** — 페이스 멈춤

사용자에게 묻기:

> "회의록 도착했고, 사실(이름·날짜·결정)이 원본과 맞나요? 됐으면 `/setup-meeting-style` 로 본인 양식을 정해 봐요 🐾"

### 7. 페어 시연 (step 6 — 5분 + 5분)

> "페어와 각자 본인 봇으로 *본인 샘플(또는 가상 메모)* 을 한 번씩 정리해 보세요.
>  결과 공유는 *회의록 캡처만* — **회의 본문(원본 메모)은 절대 공유하지 마세요**.
>  실제 회의 메모에는 동료 이름·결정 같은 민감 정보가 들어 있을 수 있어요."

## 절대 안 할 것

- 사용자의 *실제 회의 메모* 를 채팅으로 받아 갱이가 들여다보지 않기 — 시연은 *샘플* 로.
- 페어 시연 시 *회의 본문 교환* 권하지 않기 — 캡처만.
- 회의록 사실이 틀어졌는데 "원래 그래요" 로 넘기지 않기 — 사실 보존이 W5 핵심이라 반드시 점검.

## FAQ

**Q. 회의록이 안 와요.**
A. 세 가지 체크: (1) 봇 살아있나 (`ps -p $(cat .bot.pid)`) (2) W5 코드 반영하려고 봇 *재시작* 했나 (`/welcome-w5` step 6) (3) `tail -f /tmp/doppel-bot.log` 에 `notes_command` 또는 `meeting_trigger_matched` 찍히나.

**Q. 짧은 메모인데 회의록 모드로 안 가요.**
A. 600자 미만·한 줄짜리는 자동 인식이 안 돼요 (평범한 수다와 구분). 짧은 메모는 앞에 `/notes` 를 붙이거나 "회의록 정리해줘" 를 함께 보내세요.

**Q. 회의록이 중간에 잘려요.**
A. 안 잘려요 — *여러 메시지로 분할 발송* 돼요. (1/3),(2/3),(3/3) 순서대로 와요. 만약 진짜 잘렸으면(끝에 "…길이 한도") 로그 확인해 주세요.
