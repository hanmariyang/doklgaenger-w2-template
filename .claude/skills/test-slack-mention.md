# /test-slack-mention — 페어 시연

> 사용 시점: W4 step 5. `/setup-slack-mention` 첫 시연 OK 후.
>
> 모두가 같은 속도로 5+5 분 진행하는 페어 실습입니다.

## 갱이가 할 일

### 1. 페어 안내

사용자에게 보여 줄 룰:

```
페어 시연 룰:
  - 같은 슬랙 워크스페이스 안에 페어 두 명이 함께 있어야 해요.
    └─ 회사 슬랙이면 자동으로 OK.
    └─ 본인 워크스페이스 신설한 분은 페어를 *게스트 초대*:
       워크스페이스 설정 → 멤버 → "초대" → 페어 이메일.
  - 또는 *본인이 본인 두 슬랙 계정* (개인 워크스페이스 ↔ 회사 워크스페이스 본인) 으로 시연 가능.
  - 또는 *본인 셀프 멘션* (본인이 본인을 멘션) 으로도 충분 — 같은 워크스페이스 안.
```

### 2. step 1 — 페어 A 가 페어 B 멘션 (5 분)

사용자 A 에게 안내:

> "페어 B 를 멘션해서 *실제 답변이 필요한 짧은 질문* 1개 보내 보세요.
>   예: \"@페어B 이번 주 회고 자료 어디 있어요?\"
>
> 5~30 초 후 페어 B 의 텔레그램에 답변 초안 도착해야 해요.
>
> *페어 B 는 도착한 답변 초안을 검토*하고, 마음에 들면 *손으로* 슬랙에 답변. 마음에 안 들면 수정 후 답변. 봇은 슬랙에 안 답해요 — 본인 손에 맡김."

### 3. **여기까지 됐나요?** — 첫 페이스 멈춤

사용자에게 묻기:

> "페어 B 의 텔레그램에 답변 초안 도착했나요? 도착했으면 다음. 안 도착했으면 step 4 (디버그).
>
> 도착했지만 *페르소나 톤이 어색한* 답변이 왔다 → `/tune-persona` 로 W2 페르소나 튜닝 (W4 답변 초안에도 즉시 반영)."

### 4. step 2 — 페어 B 가 페어 A 멘션 (5 분)

대칭 시연. 같은 흐름.

### 5. 디버그 — 도착 안 함 시

사용자에게 6 가지 체크:

```
1) 봇 살아있나? → ps -p $(cat .bot.pid) 2>/dev/null
2) 슬랙 연결됐나? → tail -30 /tmp/doppel-bot.log | grep -E 'slack_bot_auth_test|slack_socket'
3) 봇이 그 채널에 *초대*되어 있나? → 슬랙 채널에서 /invite @도클갱어 답변봇
4) slack_user_id 정확? → grep slack_user_id config/slack_mention.yml
5) 채널 allowlist 안 걸렸나? → grep channel_allowlist config/slack_mention.yml (빈 list 면 전체 허용)
6) 키워드 필터 안 걸렸나? → grep keyword_filter config/slack_mention.yml (빈 list 면 전체 허용)
```

`/diagnose` 와 같이 보면 더 정확.

### 6. 시연 후 조정

답변 초안의 톤이 아쉽다면:

- *페르소나 자체* 튜닝 → `/tune-persona`
- *답변 길이/형식* 조정 → 페르소나 시스템 프롬프트에 한 줄 추가 (예: "답변은 3 줄 이하")
- *너무 많은 멘션이 들어옴* → `/setup-slack-filter` 로 채널·키워드 좁히기

### 7. **결승선 안내**

사용자에게 안내:

```
W4 결승선 — 손에 들고 갈 것 2 개:
  1) 멘션 → 답변 초안 도착 캡처 1 장 (페르소나 톤 살아있는 컷)
  2) 본인 yml 양식 1 줄 메모 (예: "전체 채널, 키워드 ['질문','?'], context 3")

W5 예고:
  W5 는 회의록 (텔레그램에 회의 텍스트 paste → 페르소나 요약).
  W7 부터 *본인 톤*으로 페르소나 교체 — 도구는 같고 페르소나만 바꿔요.
```

## 절대 안 할 것

- 페어를 *대신해서* 멘션 보내지 않기 — 본인 손으로.
- 답변 초안을 *대신 슬랙에 붙여 넣지* 않기 — 사용자 검토 후 *본인 손*으로.
- *민감 채널*에서 시연 권하지 않기 — `#random` 같이 부담 적은 채널 권장.

## FAQ

**Q. 페어가 다른 워크스페이스에 있어요.**
A. 본인 셀프 시연 (본인이 본인을 멘션) 으로 충분. *답변 초안*이 텔레그램에 도착하면 성공.

**Q. 답변 초안이 너무 길어요.**
A. 페르소나 시스템 프롬프트에 "답변은 3 줄 이하" 같은 제약 한 줄 추가. `/tune-persona` 호출.

**Q. 멘션 안 한 채널에서도 답변 초안이 와요.**
A. message 이벤트가 채널 전체에서 들어오기 때문. 본인 user_id 멘션이 *포함된* 메시지만 트리거 — yml `slack_user_id` 정확히 박혔는지 다시 확인.
