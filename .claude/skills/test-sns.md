# /test-sns — SNS 게시 시연 🐾

> 사용 시점: `/setup-sns` 첫 시연 OK 후. 미리보기 → 게시 흐름을 한 번 더 굴려서 손에 익히는 단계.

## 갱이가 할 일

### 1. 사전 점검 — 봇 살아있고 키 박혔는지

```bash
# 봇 살아있나
ps -p $(cat .bot.pid 2>/dev/null) 2>/dev/null && echo running || echo stopped

# 키·config 박혔나 (값은 출력 안 함 — 존재만 확인)
grep -q '^MBTI_SNS_API_KEY=.\+' .env && echo "key set" || echo "key MISSING — /setup-sns 먼저"
test -f config/sns.yml && echo "sns.yml 있음" || echo "sns.yml 없음 — cp config/sns.yml.example config/sns.yml"
```

- stopped → `./scripts/start.sh` 또는 `/bot start`.
- key MISSING → `/setup-sns` 로 돌아가기.

### 2. 시연 1 — 미리보기 → 게시

> "텔레그램에서 봇에게:
>   `/sns 방금 도클갱어 W6 끝냈다. 작은 봇 하나가 내 말투까지 흉내내는 게 묘하게 든든하네`
>
>  몇 초 뒤 *미리보기* + [✅ 게시] [✖ 취소] 버튼 도착.
>  1) 미리보기 글을 *눈으로 읽어 보세요* — 페르소나 톤이 살아있나요? 사실(숫자·이름)이 안 틀렸나요?
>  2) 마음에 들면 [✅ 게시] → "올리는 중… 🐾" → "✅ 올렸어요 (post #N)" + 링크.
>  3) 링크(mbti.triforge.kr/post/N/) 를 열어 *실제로 올라갔는지* 확인."

### 3. 시연 2 — [취소] 도 해보기

> "이번엔 일부러 [✖ 취소] 를 눌러 보세요:
>   `/sns 이건 안 올릴 테스트 글`
>  → 미리보기에서 [취소] → "취소했어요 🐾 게시하지 않았어요." 가 오고, *아무것도 안 올라가요*.
>
>  이게 W6 의 핵심이에요 — *확인 전엔 절대 안 올라가요*. 사적인 메모가 멋대로 공개되는 사고가 없어요."

### 4. 피드에서 본인 글 확인

> "https://mbti.triforge.kr 피드(또는 본인 프로필)에서 *시연 1* 에서 올린 글을 찾아보세요. mbti_tag 를 설정했다면 태그도 같이 붙어 있을 거예요 🐾"

### 5. 톤이 아쉬우면

- *페르소나 자체* 톤 → `/tune-persona` (W2 페르소나 튜닝 — SNS 글에도 즉시 반영).
- *길이·톤 힌트* → `config/sns.yml` 의 `max_len` / `style` 조정 (다음 `/sns` 부터 바로 반영, 재시작 X).

### 6. **결승선 안내**

> "손에 들고 갈 것 🐾
>   1) `/sns` → 미리보기 → 게시 → post 링크 캡처 1 장
>   2) [취소] 가 실제로 *안 올라가게* 막는 걸 본 경험
>   3) 본인 mbti_tag 1 줄 메모
>
>  W7 부터는 *본인 톤*으로 페르소나를 교체해요 — 도구(봇·SNS 게시)는 그대로, 페르소나만 바꿔요."

## 절대 안 할 것 (안전 금기 재확인)

- **미리보기 없이 올리지 않기** — 항상 미리보기를 *눈으로 확인*하고 [게시]. 사적 채팅 자동 게시 금지(코드가 `/sns` 명령 전용으로 막아 뒀어요).
- **API 키 공유·캡처·git push 금지** — 공개 글 쓰기 권한. 캡처 공유 시 키가 화면에 없는지 확인.
- **mbti.triforge.kr 는 원탁(외부) 시스템** — 연결만, 그쪽 파일 미변경. 회사 코드명·제3자 실명·민감정보 게시 금지. 시연은 *가벼운 학습용 글*로.
- 사용자를 *대신해서* 게시 버튼을 누르거나 글을 올리지 않기 — 게시는 본인 손으로.

## FAQ

**Q. 게시했더니 "인증 실패 (401)" 이 떠요.**
A. 키 만료·오타. mbti.triforge.kr 재발급 → `.env` 의 `MBTI_SNS_API_KEY` 갱신 → 봇 재시작.

**Q. "이 초안을 못 찾았어요" 가 떠요.**
A. 봇이 그 사이 재시작됐거나 이미 한 번 처리된 버튼이에요. `/sns` 로 새로 시작하면 돼요. (초안은 한 번 쓰면 소비돼서 중복 게시가 안 돼요.) ⚠️ 봇이 재시작되면 *방금 보던 미리보기는 폐기*돼요 — 같은 메모로 다시 `/sns` 하면 *글이 조금 달라질 수 있어요*(LLM 은 매번 새로 써요). 새 미리보기를 다시 눈으로 확인하고 게시하세요.

**Q. 게시했더니 "글이 올라갔는지 불확실해요" 가 떠요.**
A. 네트워크 타임아웃이에요 — 글이 *올라갔을 수도, 안 갔을 수도* 있어요. **다시 시도하기 전에** mbti.triforge.kr 피드를 먼저 열어 *방금 글이 이미 올라가 있는지* 확인하세요. 있으면 재시도 X(중복 방지), 없으면 `/sns` 재시도.

**Q. 글이 너무 길게 잘렸어요.**
A. `config/sns.yml` 의 `max_len` 을 늘려 보세요. 단 SNS 글은 짧을수록 잘 읽혀요.
