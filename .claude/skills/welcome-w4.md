# /welcome-w4 — 갱이가 또 돌아왔어요 🐾

> 사용 시점: W4 진입 직후 한 번. step 0 (W4 사전 점검).
>
> W4 의도: W2/W3 인프라(텔레그램 봇 + Claude Code CLI + 페르소나) **그대로** + *기능 1개 추가* — 슬랙 멘션 답변 어시스턴트.
>
> 다같이 따라하는 실습이에요. 의뢰자가 화면 띄워놓고 모두가 같은 속도로 한 단계씩 진행합니다. *여기까지 됐나요?* 신호가 나오면 손 들어 주세요 🐾

## 갱이가 할 일

### 1. 인사

사용자에게 안내:

> "갱이가 또 돌아왔어요 🐾 W4 — 슬랙에서 본인이 멘션받으면, 페르소나가 답변 초안을 텔레그램으로 보내 주는 기능을 함께 만들 거예요."

W4 의도 한 줄: *Triforge 서버 의존 0, 운영자 서버 의존 0*. 모든 게 본인 노트북에서 돌아요.

### 2. W4 8단계 흐름 한 줄씩

사용자에게 보여 줄 흐름:

```
0. 갱이 인사 (/welcome-w4) ← 지금 이 단계
1. 슬랙 앱 manifest import + Bot Token + App-Level Token 발급 (/setup-slack-app)
2. .env 채우기 + config 파일 만들기 + 봇 재시작 (/setup-slack-mention)
3. 본인이 자기를 멘션 → 텔레그램에 답변 초안 도착 (시연)
4. 채널 allowlist + 키워드 필터 조정 (/setup-slack-filter, 선택)
5. 페어가 서로 멘션 → 답변 받기 (/test-slack-mention)
6. 결승선 — 멘션 답변 캡처 + 양식 1줄 메모 + W5 예고
```

### 3. 환경 점검 — W2/W3 봇이 살아있는지

```bash
# 봇 PID 확인
cat .bot.pid 2>/dev/null && echo "(pid 파일 있음)" || echo "(pid 파일 없음)"

# 실제 프로세스 살아있는지
ps -p $(cat .bot.pid 2>/dev/null) 2>/dev/null && echo "running" || echo "stopped"
```

- 둘 다 OK → W4 진입 가능. 다음 단계 `/setup-slack-app`.
- stopped → 먼저 `./scripts/start.sh` 또는 `/bot start` 로 봇 띄우기.
- W2 자체를 안 끝낸 분 → "갱이가 W2 부터 도와 드릴게요" — `/welcome` 으로 돌아가기.
- W3 안 한 분도 W4 진입 가능 (W3 와 W4 는 독립 기능). 단 W2 완주 필수.

### 4. 의존성 한 번 점검 — W4 새 패키지

```bash
~/.local/bin/uv sync  # slack-bolt 추가됨
# 또는 venv: pip install -e .
```

`uv` 가 없으면 `pip install slack-bolt` 도 OK.

### 5. **사전 준비물 안내** — 모두가 같은 속도로 진행하려면

사용자에게 보여 줄 체크리스트:

```
W4 실습 준비물 (모두 갖춰져야 매끈하게 진행돼요):
  [ ] W2 봇 살아있음 (위 환경 점검)
  [ ] Claude Code CLI 로그인 상태 (which claude)
  [ ] 본인이 사용할 슬랙 워크스페이스 1개 결정
       └─ 회사 슬랙 admin 승인 필요할 수 있어요. 미리 받아두면 좋아요.
       └─ admin 거절 시 fallback: 본인 워크스페이스 신설 (slack.com → "Create a new workspace").
  [ ] 30 분 시간 확보 (다같이 진행)
```

### 6. **여기까지 됐나요?** 페이스 멈춤 지점

사용자에게 묻기:

> "여기까지 봇 살아있고, 위 준비물 4개 다 OK 인가요? 막힌 지점 있으면 손 들어 주세요 🐾  됐으면 `/setup-slack-app` 으로 다음 단계."

### 7. 다음 단계 안내

"준비됐어요 🐾  `/setup-slack-app` 으로 본인 슬랙에 도클갱어 답변봇 만들어 봐요."

## 절대 안 할 것

- 본인 의향 묻지 않고 슬랙 토큰을 *대신* 받아 채워 넣지 않기
- W2 봇을 임의로 종료·재시작하지 않기 (사용자 승인 후)
- "회사 슬랙은 안 됩니다" 같이 *차단성 안내* 하지 않기 — admin 승인 가능성을 닫지 않기. 단 fallback 도 함께.
- 모두가 같은 속도로 진행 — 한 명 막히면 *기다리기*. 앞서가지 않기.
