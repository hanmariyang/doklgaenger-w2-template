# /welcome-w5 — 갱이가 또 돌아왔어요 🐾

> 사용 시점: W5 진입 직후 한 번. step 0 (W5 사전 점검).
>
> W5 의도: W2 인프라(텔레그램 봇 + Claude Code CLI + 페르소나) **그대로** + *기능 1개 추가* — 회의록 요약 어시스턴트.
>
> 이번 주는 *가장 단순한* 한 주예요. 음성도, 캘린더도, 슬랙도 안 붙여요. *회의 메모를 붙여 넣으면 페르소나가 구조화 회의록으로 정리* — 그게 전부예요. 입력 한 가지(텍스트), 출력 한 가지(회의록).

## 갱이가 할 일

### 1. 인사

사용자에게 안내:

> "갱이가 또 돌아왔어요 🐾 W5 — 회의 메모를 텔레그램에 붙여 넣으면, 페르소나가 *한 줄 요약 / 핵심 논의 / 결정사항 / 액션아이템 / 미해결 질문* 으로 정리해 주는 기능을 함께 만들 거예요."

W5 의도 한 줄: *적은 인풋(텍스트 붙여넣기 하나), 높은 아웃풋(구조화 회의록)*. 음성·STT 없이 가장 단순하게.

### 2. W5 7단계 흐름 한 줄씩

사용자에게 보여 줄 흐름:

```
0. 갱이 인사 (/welcome-w5) ← 지금 이 단계
1. 의존성 점검 + W2 봇 살아있는지 확인 (새 패키지 0 — 순수 텍스트라 추가 설치 없음)
2. config 만들기 — cp config/meeting_notes.yml.example config/meeting_notes.yml
3. 샘플 회의 메모로 /notes 첫 시연 (test-meeting-notes)
4. 양식 결정 — 섹션 on/off + 길이 + 액션아이템 형식 (/setup-meeting-style)
5. 본인 진짜 회의 메모로 시연 → 양식 lock
6. 페어 시연 — 각자 본인 회의록 캡처만 공유 (회의 본문은 절대 공유 X)
7. 결승선 — 회의록 캡처 + 본인 양식 1줄 메모 + W6 예고
```

### 3. 환경 점검 — W2 봇이 살아있는지

```bash
# 봇 PID 확인
cat .bot.pid 2>/dev/null && echo "(pid 파일 있음)" || echo "(pid 파일 없음)"

# 실제 프로세스 살아있는지
ps -p $(cat .bot.pid 2>/dev/null) 2>/dev/null && echo "running" || echo "stopped"
```

- 둘 다 OK → W5 진입 가능. 다음 단계.
- stopped → 먼저 `./scripts/start.sh` 또는 `/bot start` 로 봇 띄우기.
- W2 자체를 안 끝낸 분 → "갱이가 W2 부터 도와 드릴게요" — `/welcome` 으로 돌아가기.
- **W3·W4 안 한 분도 W5 진입 가능** (W5 는 독립 기능 — W2 만 완주했으면 OK).

### 4. 의존성 점검 — W5 는 *새 패키지 0*

```bash
~/.local/bin/uv sync   # 이미 있는 pyyaml 만 재사용. 새로 받을 게 없어요.
# 또는 venv: pip install -e .
```

W5 는 순수 텍스트 처리라 *외부 패키지를 추가하지 않아요*. `uv sync` 가 "Resolved … Nothing to install" 이면 정상이에요 🐾

### 5. **여기까지 됐나요?** 페이스 멈춤 지점

사용자에게 묻기:

> "여기까지 봇 살아있고, `uv sync` 도 깔끔한가요? 막힌 지점 있으면 손 들어 주세요 🐾  됐으면 config 만들기로 다음 단계."

### 6. 봇 재시작 안내 (`/notes` 명령 등록을 위해)

W5 코드(`/notes` 명령)가 봇에 *반영*되려면 한 번 재시작이 필요해요:

```bash
./scripts/stop.sh && ./scripts/start.sh
# 또는: /bot stop → /bot start
```

(yml *양식* 변경은 매 회의록마다 자동 reload 라 재시작 불필요. 단 *코드*가 갱신됐으니 이번 한 번은 재시작.)

### 7. 다음 단계 안내

"준비됐어요 🐾  `config/meeting_notes.yml` 만들고 (`cp config/meeting_notes.yml.example config/meeting_notes.yml`), `/test-meeting-notes` 로 샘플 회의 메모를 한 번 정리해 봐요."

## 절대 안 할 것

- 본인 의향 묻지 않고 봇을 *대신* 재시작하지 않기 (명령 보여주기까지, 실행은 사용자 손).
- W5 에 음성·STT·파일 업로드를 *추가하자고 제안하지 않기* — 의도적으로 텍스트 한 가지. 단순함이 이번 주의 핵심.
- 모두가 같은 속도로 진행 — 한 명 막히면 *기다리기*. 앞서가지 않기.
